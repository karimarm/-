ТЕХНИЧЕСКОЕ ЗАДАНИЕ
на разработку программного обеспечения для обработки библиографических списков
"БиблиоАналитика"

1. ОБЩИЕ СВЕДЕНИЯ

1.1. Наименование программы
    Полное наименование: Программное обеспечение для обработки библиографических списков "БиблиоАналитика"
    Краткое наименование: ПО "БиблиоАналитика"

1.2. Основание для разработки
    Необходимость автоматизации процессов обработки, анализа и форматирования библиографических списков для научных работ.

1.3. Назначение и цели создания программы
    Программа предназначена для автоматизации работы с библиографическими списками, включая распознавание элементов библиографических ссылок, проверку соответствия списка критериям, поиск источников в онлайн библиотеках и форматирование согласно стандартам цитирования.
    
    Основные цели создания программы:
    - Упрощение работы с библиографическими списками для студентов и научных сотрудников
    - Автоматизация форматирования библиографических ссылок согласно различным стандартам
    - Анализ библиографического списка на соответствие заданным критериям
    - Помощь в поиске и добавлении новых библиографических источников

1.4. Целевая аудитория
    - Студенты
    - Аспиранты
    - Научные сотрудники
    - Преподаватели
    - Редакторы научных изданий

2. ТРЕБОВАНИЯ К ПРОГРАММЕ

2.1. Требования к функциональным характеристикам

2.1.1. Модуль распознавания элементов библиографических ссылок
    - Автоматическое распознавание элементов (авторы, название, год издания, издательство и др.) из введенного текста
    - Поддержка русского и английского языков
    - Распознавание различных типов источников: книги, статьи, журналы, веб-ресурсы и т.д.
    - Обязательная поддержка форматов ГОСТ и IEEE
    - Интеллектуальное определение формата ссылки
    - Возможность корректировки неверно распознанных элементов

2.1.2. Модуль проверки соответствия библиографического списка критериям
    - Проверка наличия источников на английском языке (количество/процент)
    - Проверка наличия источников свежее заданного года (количество/процент)
    - Проверка наличия источников ВАК, РИНЦ (количество/процент)
    - Контроль количества источников одного автора (количество/процент)
    - Настраиваемые пороговые значения для всех критериев
    - Формирование отчета о соответствии критериям
    - Визуализация результатов проверки

2.1.3. Модуль поиска источников в онлайн библиотеках
    - Интеграция с базами данных: ELibrary, Scholar.ru, Google Scholar, КиберЛенинка и др.
    - Поиск по ключевым словам, авторам, названиям
    - Фильтрация результатов по году, типу публикации, рейтингу
    - Предварительный просмотр найденных источников
    - Автоматическое добавление в библиографический список
    - Сохранение истории поисковых запросов

2.1.4. Модуль ввода библиографических ссылок
    - Ручной ввод через специализированную форму
    - Импорт из форматов .docx, .pdf
    - Прямой текстовый ввод с распознаванием
    - Копирование из буфера обмена
    - Автоматическое дополнение недостающей информации
    - Проверка корректности введенных данных

2.1.5. Модуль форматирования и экспорта
    - Поддержка стандартов цитирования (ГОСТ, APA, MLA, Harvard, IEEE и др.)
    - Экспорт в форматы .docx, .pdf
    - Предварительный просмотр сформированного списка
    - Сохранение шаблонов форматирования
    - Пакетное форматирование всего списка

2.2. Требования к надежности

2.2.1. Требования к обеспечению надежного функционирования
    - Контроль целостности данных
    - Ведение журнала операций
    - Автоматическое сохранение промежуточных результатов
    - Восстановление после сбоев
    - Проверка корректности входных данных

2.2.2. Требования к безопасности
    - Защита от несанкционированного доступа
    - Безопасный обмен данными с внешними сервисами
    - Контроль доступа к функциям программы
    - Защита персональных настроек пользователя

2.3. Требования к эргономике и технической эстетике
    - Интуитивно понятный интерфейс
    - Адаптивный дизайн
    - Поддержка тем оформления (светлая/темная)
    - Настраиваемые параметры отображения
    - Система контекстных подсказок
    - Многоязычный интерфейс (русский, английский)

2.4. Требования к информационной и программной совместимости

2.4.1. Требования к информационным структурам
    - Хранение данных в формате JSON/XML
    - Совместимость с библиографическими менеджерами (EndNote, Mendeley, Zotero)
    - Поддержка основных форматов обмена библиографическими данными

2.4.2. Требования к программным средствам
    - Язык программирования: Python
    - Платформа: Windows 10 и выше
    - Необходимое ПО: Microsoft Office (для поддержки .docx) или аналоги
    - Доступ к интернету для онлайн функций

2.4.3. Требования к аппаратным средствам
    - Процессор: не ниже Intel Core i3 или аналогичный
    - Оперативная память: не менее 4 ГБ
    - Свободное место на жестком диске: не менее 500 МБ
    - Монитор с разрешением не менее 1366x768
